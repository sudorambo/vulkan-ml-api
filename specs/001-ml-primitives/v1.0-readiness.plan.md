# Plan: v1.0 Release Readiness

**Goal**: Resolve all blocking issues identified in the final ship-readiness review so the repository can be tagged `1.0.0`.

Findings are grouped into 6 phases, ordered by dependency and severity. Each phase can be verified independently before proceeding.

---

## Phase 1: Public API Surface Fixes (CRITICAL)

These must land first because every other component (ICD, validation, tests) depends on correct header values.

### 1.1 Fix duplicate VkStructureType values (C1)

**Problem**: The four new primitive descriptor sTypes (1000559020–23) collide with dispatch info, tensor binding, dependency info, and tensor copy sTypes assigned the same numeric values.

**File**: `include/vulkan/vulkan_ml_primitives.h:69-76`

**Fix**: Reassign the colliding group to unique values:
```c
VK_STRUCTURE_TYPE_ML_GRAPH_DISPATCH_INFO_KHR  = 1000559024,
VK_STRUCTURE_TYPE_ML_TENSOR_BINDING_KHR       = 1000559025,
VK_STRUCTURE_TYPE_TENSOR_DEPENDENCY_INFO_KHR  = 1000559026,
VK_STRUCTURE_TYPE_TENSOR_COPY_KHR             = 1000559027,
```

**Ripple**: Update every `(int)X == VK_STRUCTURE_TYPE_TENSOR_COPY_KHR` comparison in `src/tensor_copy.c`, `layers/validation/tensor_validation.c`, and test files. Numeric values in tests that hard-code sType values must also be updated.

### 1.2 Rename `VK_FORMAT_R8_BOOL` to `VK_FORMAT_R8_BOOL_KHR` (H8)

**Problem**: All other extension-defined format values carry the `_KHR` suffix. This one does not, violating Vulkan naming conventions.

**File**: `include/vulkan/vulkan_ml_primitives.h:115`

**Fix**: Rename `VK_FORMAT_R8_BOOL` to `VK_FORMAT_R8_BOOL_KHR`.

**Ripple**: Update `src/internal.h:137` (`vk_ml_format_element_size` switch case), `src/feature_query.c`, and any tests referencing this enum.

### 1.3 Move `VkMLResizeModeKHR` enum to Enumerations section (M-polish)

**Problem**: `VkMLResizeModeKHR` (line 444) is defined between struct definitions instead of in the "Enumerations (T005)" section (lines 121–205).

**File**: `include/vulkan/vulkan_ml_primitives.h:444-449`

**Fix**: Move the typedef enum block to after `VkMLTensorBindingTypeKHR` (line 205), before the "Bitmask types" section.

## Phase 2: ICD Correctness Fixes (CRITICAL + HIGH)

With the header stable, fix the reference ICD implementation bugs.

### 2.1 Add missing op desc deep-copy cases (C2)

**Problem**: `op_desc_size_by_stype()` in `ml_graph.c:106-124` does not handle the 4 new descriptor types (concat, reshape, transpose, resize). `deep_copy_op_desc()` returns NULL for these, causing graph creation to fail with spurious `VK_ERROR_OUT_OF_HOST_MEMORY`.

**File**: `src/ml_graph.c:106-124`

**Fix**: Add four cases to the switch:
```c
case VK_STRUCTURE_TYPE_ML_PRIMITIVE_DESC_CONCAT_KHR:
    return sizeof(VkMLPrimitiveDescConcatKHR);
case VK_STRUCTURE_TYPE_ML_PRIMITIVE_DESC_RESHAPE_KHR:
    return sizeof(VkMLPrimitiveDescReshapeKHR);
case VK_STRUCTURE_TYPE_ML_PRIMITIVE_DESC_TRANSPOSE_KHR:
    return sizeof(VkMLPrimitiveDescTransposeKHR);
case VK_STRUCTURE_TYPE_ML_PRIMITIVE_DESC_RESIZE_KHR:
    return sizeof(VkMLPrimitiveDescResizeKHR);
```

**Note**: Reshape and transpose descriptors contain pointer members (`pOutputDimensions`, `pPermutation`) that need deep-copying. `deep_copy_op_desc` currently does a shallow `memcpy`. Add post-copy deep-copy logic for these two descriptor types, similar to `deep_copy_tensor_desc`.

### 2.2 Guard scratch size calculation against NULL desc arrays (C3)

**Problem**: The scratch size loop at `ml_graph.c:412-421` unconditionally dereferences `externalInputDescs`, `externalOutputDescs`, `constantWeightDescs`. If `externalInputCount > 0` but `pExternalInputDescriptions` was NULL (which passes the preceding guard due to the `&&` check), the loop dereferences NULL.

**File**: `src/ml_graph.c:411-421`

**Fix**: Guard each loop with a NULL check on the desc array:
```c
if (graph->externalInputDescs) {
    for (uint32_t i = 0; i < graph->externalInputCount; i++)
        totalSize += tensor_desc_size(&graph->externalInputDescs[i]);
}
```
Repeat for `externalOutputDescs` and `constantWeightDescs`.

### 2.3 Zero-initialize session struct (H2)

**Problem**: When `scratchMemory == VK_NULL_HANDLE` and `scratchMemorySize == 0`, the session's scratch fields are uninitialized (garbage from `malloc`).

**File**: `src/ml_session.c:24-52`

**Fix**: Add `memset(session, 0, sizeof(VkMLSessionKHR_T));` immediately after the `vk_ml_alloc` call (before setting any fields).

### 2.4 Validate user-provided scratch memory size (H3)

**Problem**: When user provides explicit scratch memory, there is no check that `pCreateInfo->scratchMemorySize >= g->scratchMemorySize`. VUID `VUID_SESSION_SCRATCH_SIZE` is defined but never enforced in the ICD.

**File**: `src/ml_session.c:34-37`

**Fix**: Add after the `VK_NULL_HANDLE` check:
```c
if (pCreateInfo->scratchMemorySize < g->scratchMemorySize)
    return VK_ERROR_UNKNOWN;
```

### 2.5 Add sType validation to ICD create functions (H4)

**Problem**: Neither `vkCreateTensorKHR` nor `vkCreateTensorViewKHR` validates `pCreateInfo->sType`. A struct with the wrong sType could be misinterpreted.

**Files**: `src/tensor.c:19`, `src/tensor_view.c:19`

**Fix**: Add sType checks:
- In `tensor.c` after the NULL checks: `if ((int)pCreateInfo->sType != VK_STRUCTURE_TYPE_TENSOR_CREATE_INFO_KHR) return VK_ERROR_UNKNOWN;`
- In `tensor_view.c` after the NULL checks: `if ((int)pCreateInfo->sType != VK_STRUCTURE_TYPE_TENSOR_VIEW_CREATE_INFO_KHR) return VK_ERROR_UNKNOWN;`

### 2.6 Return error for NULL tensor handle in bind (H5)

**Problem**: `vkBindTensorMemoryKHR` silently skips NULL tensor handles with `continue`. Per Vulkan convention (`vkBindBufferMemory2`), each element's handle must be valid.

**File**: `src/tensor.c:140-141`

**Fix**: Replace `continue` with `return VK_ERROR_UNKNOWN;`

### 2.7 Add `pExtents` NULL check in tensor copy stub (M3)

**Problem**: `tensor_copy.c:38` checks `pSrcOffsets` and `pDstOffsets` for NULL when `dimensionCount > 0`, but never checks `pExtents`.

**File**: `src/tensor_copy.c:38`

**Fix**: Change the condition to:
```c
if (r->dimensionCount > 0 && (!r->pSrcOffsets || !r->pDstOffsets || !r->pExtents))
    return;
```

### 2.8 Remove `(int)` cast for sType comparisons (M2)

**Problem**: `(int)` cast on `VkStructureType` (which is `uint32_t`) is fragile. If a sType value exceeds `INT_MAX` this becomes undefined behavior.

**Files**: `src/tensor_copy.c:24,36`, `src/ml_dispatch.c:21`

**Fix**: Cast to `(uint32_t)` or compare directly without cast, consistent with `vk_ml_format_element_size` in `internal.h`.

## Phase 3: Validation Layer Fixes (CRITICAL + HIGH)

### 3.1 Fix integer overflow in tensor view range check (C4)

**Problem**: `pDimensionOffsets[i] + pDimensionSizes[i]` wraps `uint32_t` when offset is near `UINT32_MAX`, bypassing the bounds check.

**File**: `layers/validation/tensor_validation.c:117`

**Fix**: Rewrite as overflow-safe subtraction:
```c
if (pCreateInfo->pDimensionSizes[i] > tensor->dimensions[i] ||
    pCreateInfo->pDimensionOffsets[i] > tensor->dimensions[i] - pCreateInfo->pDimensionSizes[i])
    return VK_FALSE;
```

### 3.2 Fix stack buffer overflow in DFS cycle detection (C5)

**Problem**: `color` array is stack-allocated at `VK_ML_REF_MAX_ML_GRAPH_NODE_COUNT` (256) elements, but `nodeCount` is bounded by caller-supplied `props->maxMLGraphNodeCount` which could exceed 256.

**File**: `layers/validation/graph_validation.c:66-74`

**Fix**: Add a guard before the DFS block:
```c
if (pCreateInfo->nodeCount > VK_ML_REF_MAX_ML_GRAPH_NODE_COUNT)
    return VK_FALSE;
```
This is safe for 1.0 since the reference ICD's max is 256. A dynamic allocation approach can follow post-1.0 if needed.

### 3.3 Add `pExtents` NULL check in copy validation (H6)

**Problem**: `tensor_validation.c:169-179` checks `pSrcOffsets` and `pDstOffsets` for NULL but not `pExtents`.

**File**: `layers/validation/tensor_validation.c:174-178`

**Fix**: Add after line 178:
```c
if (region->dimensionCount > 0 && !region->pExtents)
    return VK_FALSE;
```

### 3.4 Add activation descriptor validation in graph per-node check (H7)

**Problem**: The `VK_STRUCTURE_TYPE_ML_PRIMITIVE_DESC_ACTIVATION_KHR` case in `graph_validation.c:99-100` is a no-op `break`.

**File**: `layers/validation/graph_validation.c:99-100`

**Fix**: Implement `vk_ml_validate_activation_desc()`:
- Declare in `vk_ml_validation.h`
- Implement in `graph_validation.c`: validate `activationType` enum range, `param0`/`param1` finiteness, and Leaky ReLU/Clamp parameter constraints
- Wire into the per-node switch

### 3.5 Reject unknown sTypes in graph per-node validation (M1)

**Problem**: `default: break;` in the per-node descriptor switch silently accepts unknown sTypes.

**File**: `layers/validation/graph_validation.c:111-112`

**Fix**: Change to `default: return VK_FALSE;`

### 3.6 Add NULL handle checks for copy src/dst tensors (M3-val)

**Problem**: `vk_ml_validate_tensor_copy` checks `srcTensor == dstTensor` but not either handle individually for `VK_NULL_HANDLE`.

**File**: `layers/validation/tensor_validation.c:158-160`

**Fix**: Add before the same-tensor check:
```c
if (pCopyInfo->srcTensor == VK_NULL_HANDLE || pCopyInfo->dstTensor == VK_NULL_HANDLE)
    return VK_FALSE;
```

### 3.7 Add concat/reshape/transpose/resize cases to graph validation switch (M1-ext)

**Problem**: The per-node validation switch in `graph_validation.c:83-113` does not handle the 4 new descriptor types. With 3.5 applied (default returns `VK_FALSE`), graphs using these operations would be rejected.

**File**: `layers/validation/graph_validation.c:83-113`

**Fix**: Add cases for the 4 new sTypes. Concat/resize can validate via `vk_ml_validate_primitive_desc` (already covers them). Wire into the switch like the existing types.

## Phase 4: Infrastructure and Build Fixes (HIGH + MEDIUM)

### 4.1 Fix clang-tidy config override in CMakeLists.txt (H9)

**Problem**: `CMAKE_C_CLANG_TIDY` passes explicit `-checks=...` that overrides the `.clang-tidy` config file, missing its exclusions (`-readability-magic-numbers`, `-readability-function-cognitive-complexity`, `-misc-no-recursion`).

**File**: `CMakeLists.txt:186-191`

**Fix**: Remove the `-checks=...` argument and use `--config-file`:
```cmake
set(CMAKE_C_CLANG_TIDY
    ${CLANG_TIDY_EXE};
    --config-file=${CMAKE_CURRENT_SOURCE_DIR}/.clang-tidy;
    -header-filter=${CMAKE_CURRENT_SOURCE_DIR}/include/.*
)
```

### 4.2 Fix pkg-config `libdir` to use `CMAKE_INSTALL_LIBDIR` (M5)

**Problem**: `libdir=${prefix}/lib` is hardcoded, breaking `lib64` systems (Fedora, Arch).

**File**: `cmake/vk_ml_primitives.pc.in:3-4`

**Fix**:
```
libdir=@CMAKE_INSTALL_FULL_LIBDIR@
includedir=@CMAKE_INSTALL_FULL_INCLUDEDIR@
```

### 4.3 Add CMake `NAMESPACE` to exported targets (M6)

**Problem**: `install(EXPORT)` has no `NAMESPACE`, so imported target is `vk_ml_primitives` instead of `vk_ml_primitives::vk_ml_primitives`. Typos in `target_link_libraries` silently pass configure.

**File**: `CMakeLists.txt:85-88`

**Fix**: Add `NAMESPACE vk_ml_primitives::` to the export install:
```cmake
install(EXPORT vk_ml_primitivesTargets
    FILE vk_ml_primitivesTargets.cmake
    NAMESPACE vk_ml_primitives::
    DESTINATION ${CMAKE_INSTALL_LIBDIR}/cmake/vk_ml_primitives
)
```

Update `cmake/vk_ml_primitivesConfig.cmake.in` to reference `vk_ml_primitives::vk_ml_primitives`.

### 4.4 Add missing entries to `.gitignore` (M7, M8)

**Problem**: Doxygen output `docs/html/` and preset build dirs `build-release/`, `build-ci/` are not gitignored.

**File**: `.gitignore`

**Fix**: Append:
```
# Doxygen output
docs/html/

# CMake preset build directories
build-*/
```

### 4.5 Fix CI Vulkan SDK Ubuntu codename (M9)

**Problem**: `lunarg-vulkan-noble.list` is hardcoded for Ubuntu 24.04. When `ubuntu-latest` updates, this breaks.

**File**: `.github/workflows/ci.yml:36`

**Fix**: Use dynamic codename detection:
```yaml
sudo wget -qO /etc/apt/sources.list.d/lunarg-vulkan.list \
  "https://packages.lunarg.com/vulkan/lunarg-vulkan-$(lsb_release -cs).list"
```
Apply to both the `build` and `static-analysis` jobs.

### 4.6 Add clang-format check to CI (M10)

**Problem**: `.clang-format` exists but CI never verifies formatting compliance.

**File**: `.github/workflows/ci.yml` (static-analysis job)

**Fix**: Add a step:
```yaml
- name: Check formatting
  run: |
    find src include layers examples -name '*.c' -o -name '*.h' | \
      xargs clang-format --dry-run --Werror
```

### 4.7 Align compiler versions between README and CONTRIBUTING (M-doc)

**Problem**: README says GCC 11+, Clang 14+, MSVC 2022+. CONTRIBUTING says GCC 8+, Clang 6+, MSVC 2019+.

**Files**: `README.md:97-99`, `CONTRIBUTING.md:9`

**Fix**: Align both to GCC 11+, Clang 14+, MSVC 2022+ (the stricter set, matching `-Wpedantic -Werror` requirements with C17).

### 4.8 Remove `-DCMAKE_CXX_COMPILER` from CI for C-only project (L-CI)

**Problem**: CI passes `-DCMAKE_CXX_COMPILER=${{ matrix.cxx }}` but the project is `LANGUAGES C` only. Unnecessary and potentially confusing.

**File**: `.github/workflows/ci.yml:55-56`

**Fix**: Remove the `-DCMAKE_CXX_COMPILER` line from the CI configure step.
