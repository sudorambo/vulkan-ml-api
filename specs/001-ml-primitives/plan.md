# Implementation Plan: H11 — Pooling Validation Fix for Global Average Pool

**Branch**: `001-ml-primitives` | **Date**: 2026-03-06 | **Spec**: `spec/VK_KHR_ml_primitives.adoc`
**Input**: Review finding H11 from `specs/001-ml-primitives/review-findings.md:394-398`

## Summary

Fix a correctness bug where the validation layer (`vk_ml_validate_pooling_desc`) unconditionally rejects `windowWidth == 0` and `strideX == 0` for all pool types, including `VK_ML_OPERATION_TYPE_GLOBAL_AVERAGE_POOL_KHR`. For global average pooling, window size and stride are semantically irrelevant — the entire spatial extent is pooled. The ICD layer already correctly gates these checks. The fix aligns the validation layer logic with the ICD and adds a positive test and updates the existing CTS workaround.

## Analysis

### Root Cause

When `vk_ml_validate_pooling_desc` was implemented, the window/stride checks were placed unconditionally before the pool type switch. The ICD layer (`vk_ml_validate_primitive_desc`) correctly used `if (d->poolType == MAX_POOL || d->poolType == AVG_POOL)` to gate these checks, but this conditional was not replicated in the validation layer.

### Current behavior (buggy)

`vk_ml_validate_pooling_desc` rejects any pooling descriptor with `windowWidth == 0` or `strideX == 0`, regardless of pool type:

```c
/* VUID_POOL_WINDOW */
if (desc->windowWidth == 0 || desc->windowHeight == 0)
    return VK_FALSE;
/* VUID_POOL_STRIDE */
if (desc->strideX == 0 || desc->strideY == 0)
    return VK_FALSE;
```

The CTS test at `test_ml_graph.c:686` works around this by setting `windowWidth = 1, windowHeight = 1` for global average pool.

### Target behavior (fixed)

Window/stride checks only apply to `MAX_POOL_2D` and `AVERAGE_POOL_2D`. Global average pool ignores these fields. The validation layer should match the ICD:

```c
/* VUID_POOL_TYPE — check type first */
switch (desc->poolType) { ... }

/* VUID_POOL_WINDOW / VUID_POOL_STRIDE — only for windowed pool types */
if (desc->poolType != VK_ML_OPERATION_TYPE_GLOBAL_AVERAGE_POOL_KHR) {
    if (desc->windowWidth == 0 || desc->windowHeight == 0)
        return VK_FALSE;
    if (desc->strideX == 0 || desc->strideY == 0)
        return VK_FALSE;
}
```

### Test plan

1. **New positive test** (`test_descriptor_validation.c`): `test_valid_global_average_pool` — creates a pooling descriptor with `poolType = GLOBAL_AVERAGE_POOL`, `windowWidth = 0`, `windowHeight = 0`, `strideX = 0`, `strideY = 0`. Asserts `vk_ml_validate_pooling_desc` returns `VK_TRUE`.

2. **CTS test update** (`test_ml_graph.c`): Update `test_single_node_global_avg_pool` to use `windowWidth = 0, windowHeight = 0` instead of the workaround values of `1, 1`. This tests the full stack (validation + ICD) with the correct semantics.

3. **Preserve existing negative tests**: `test_pool_zero_window` and `test_pool_zero_stride` in `test_descriptor_validation.c` already use `MAX_POOL_2D` as pool type, so they remain valid negative tests.

4. **Build + test**: `cmake --build build` with zero warnings, `ctest --output-on-failure` with 13/13 pass.
